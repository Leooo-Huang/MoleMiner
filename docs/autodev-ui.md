# MoleMiner — Web 可视化界面 UI 设计

> 新增功能 | 生成时间：2026-04-04

## 设计原则

1. **数字孪生风格** — 深色背景 + 发光元素 + 科技感，配合 3D 数字地球
2. **信息密度优先** — 搜索工具用户要看到数据，不是装饰
3. **CLI 伴侣** — 不是独立 Web 应用，是 CLI 输出的可视化增强

## 信息架构

```
moleminer web → 浏览器打开
  │
  ├── Sidebar (左侧导航，固定)
  │   ├── 搜索 → #/
  │   ├── 搜索源 → #/sources
  │   └── 设置 → #/settings
  │
  ├── #/ (首页 = 搜索历史)
  │   ├── 搜索输入框 (顶栏固定)
  │   └── 历史搜索列表
  │
  ├── #/search/:id (搜索结果)
  │   ├── 搜索输入框 (顶栏固定)
  │   ├── 搜索元信息 (查询/时间/源/结果数)
  │   ├── 视图切换 [列表 | 地球]
  │   ├── ListView (默认)
  │   │   ├── 筛选栏 (source / type)
  │   │   └── 结果卡片列表
  │   └── GlobeView
  │       ├── 3D 数字地球 (带标记点)
  │       └── 侧边面板 (选中标记的详情)
  │
  ├── #/sources (搜索源管理)
  │   ├── API 源卡片网格 (toggle 启用/禁用)
  │   └── 认证源卡片网格 (toggle 触发 QR 登录)
  │
  ├── #/settings (设置)
  │   ├── AI 引擎信息 (provider/model/profile)
  │   └── 搜索默认值 (maxRounds/timeout)
  │
  └── LoginModal (全局覆盖层，按需弹出)
      ├── 搜索前自动检测需登录的源 → 队列式弹出
      ├── Sources 页启用 auth 源时弹出
      ├── QR 码展示 + 60s 倒计时 + 过期刷新
      └── 成功/跳过 → 自动推进到下一个
```

## 用户流程

### 流程 1：查看历史搜索结果
1. `moleminer web` → 浏览器打开 `localhost:3456`
2. 首页显示搜索历史列表
3. 点击某条搜索 → 进入结果详情页（默认列表视图）
4. 点击"地球视图" → 切换到 3D 数字地球
5. 拖拽旋转地球 → 找到目标区域
6. 点击标记点 → 右侧显示该位置的结果
7. 点击结果标题 → 新标签打开原始 URL

### 流程 2：从 Web UI 发起搜索
1. 在搜索框输入查询 → 按回车或点搜索
2. 进度覆盖层出现，SSE 实时更新
3. 搜索完成 → 自动跳转到结果详情页
4. 结果自动保存到历史

### 流程 3：无地理信息降级
1. 查看搜索结果 → 结果中没有 location 字段的条目
2. 地球视图按钮显示但带提示 "(N 条有位置信息)"
3. 如果 0 条有位置信息 → 按钮置灰，tooltip 说明
4. 地球视图只显示有 location 的结果，列表视图显示全部

## 动效规范

| 场景 | 动效 | 参数 |
|------|------|------|
| 页面切换 | fade in | 200ms ease-out |
| 卡片 hover | 背景色变浅 + 微升 | 150ms ease |
| 视图切换 | crossfade | 300ms ease-in-out |
| 地球标记出现 | 脉冲 + 上浮 | 400ms ease-out |
| 进度条 | 平滑增长 | CSS transition 300ms |
| 地球自转 | 缓慢持续自转 | 0.1°/帧，用户拖拽时停止 |

## 新增组件：LoginModal（QR 登录弹窗）

> 新增时间：2026-04-15

**触发方式**：
1. **搜索前自动检测**：`handleSearch` 调用 `fetchSources()` → 过滤 `requiresAuth && !hasCredentials && isInDefaultSources` → 队列式弹出
2. **Sources 页 toggle**：启用 auth 源时 `!hasCredentials` → 单个弹出

**组件 Props**：
```typescript
interface LoginModalProps {
  platform: string;   // 'zhihu' | 'weibo' | 'xiaohongshu'
  index: number;      // 当前第几个（1-based）
  total: number;      // 总共几个需要登录
  onSuccess: () => void;
  onSkip: () => void;
}
```

**状态机**：
```
starting → qr_ready → success (1.5s 后自动 onSuccess)
                    → error (显示错误信息)
         (60s 倒计时 → 过期 → 显示"刷新"按钮)
```

**布局**：
```
┌──────────────────────────────┐
│  登录 知乎          1/3 个平台 │
├──────────────────────────────┤
│                              │
│        ┌──────────┐          │
│        │  QR 码    │          │
│        │ 248×248   │          │
│        └──────────┘          │
│                              │
│   使用 知乎 App 扫一扫登录     │
│        有效期：45s             │  ← <15s 变 warning 色
│                              │
│   [───── 跳过 ─────]         │
└──────────────────────────────┘
```

**关键实现**：
- 复用 CLI `playwrightLogin()` + `onQrReady` 回调（服务端生成 PNG data URL）
- SSE 事件流：`qr_ready` / `success` / `error` / `cancelled`
- `cancelToken` 取消：跳过/卸载时通知服务端终止 Playwright 浏览器
- `refreshCount` 状态驱动 QR 刷新（取消旧会话 → 重新 startPlatformLogin）
- unmount 时 cleanup 调用 `cancelPlatformLogin()` 释放服务端资源
